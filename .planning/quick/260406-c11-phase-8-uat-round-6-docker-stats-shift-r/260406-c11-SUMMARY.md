## Changes Delivered

### Task 1 — ServiceCard.tsx

1. **Docker stats column gap**: Added `paddingLeft: '8px'` to the RIGHT col div — Docker CPU/RAM text no longer collides with the bar number column on its left.

2. **Volume bar label**: Changed fallback from `.slice(0,4)` (producing "VOLU") to:
   - `/volume1` → `HD`
   - `/volume2`, `/volume3`, etc. → `HD2`, `HD3`, etc.
   - Other names → `.slice(0,4)`

3. **Pi-hole MEM + LOAD rows**: Added two 22px bold amber stat rows (MEM and LOAD) below the BLOCKED row in the Pi-hole left column. These use `metrics.memPercent` and `metrics.load1m` which were already parsed in the component.

4. **percentBlocked font size**: Bumped from 13px to 22px to match the visual weight of BLOCKING, QPM, MEM, and LOAD.

5. **Ubiquiti vertical bars**: Increased container height from `60px` to `90px` to fill the taller Pi-hole tile proportionally.

### Task 2 — NowPlayingBanner.tsx

6. **Rail height**: Both idle and active collapsed strips increased from `40px` to `48px`.

7. **All text to 22px**: PLEX label, "NO ACTIVE STREAMS" text, cycling stream title, and all three server stats (CPU/RAM/BW) bumped to `22px` with `fontWeight: 600` in both idle and active states.

### Task 2 — CardGrid.tsx (DownloadActivity)

8. **DOWNLOADS header**: Reduced from `22px` to `11px` proportional size — matches other small section headers.

9. **Active download layout**: Replaced title + x{count} layout with title-only (22px bold purple) + thick amber progress bar (12px, from `downloadProgress` metric). No service name or count shown.

10. **Idle state**: Removed "IDLE" text. Changed `minHeight` from `'120px'` to `'auto'` — idle state shows only the small DOWNLOADS header with no wasted space.
