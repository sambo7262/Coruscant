## New Components: Log Viewer

### LogsPage Structure (D-28 through D-33)

**Layout:** Full-height page, no card wrapper. Dark panel background (`--space-mid`). Cockpit aesthetic table — monospace, compact rows.

```
[PAGE HEADER]  "SYSTEM LOGS"  (heading role, amber)
[FILTER BAR]   [LEVEL: WARN+▾] [SERVICE: ALL▾]   [EXPORT LOGS]  [PURGE LOGS]
[LOG TABLE]
  TIME      LEVEL     SERVICE      MESSAGE
  HH:mm:ss  [WARN]   [RADARR]     Lost connection to arr endpoint
  HH:mm:ss  [ERROR]  [NAS]        DSM session expired — re-authing
[LOAD MORE / showing N entries]
```

**Table columns:**
- `TIME` — 9-char fixed-width `HH:mm:ss`, label role (13px), `--text-offwhite`
- `LEVEL` — colored chip badge, label role, uppercase: INFO (dim amber chip), WARN (amber chip), ERROR (red chip). Fixed 48px width.
- `SERVICE` — amber tag badge, label role, uppercase, max 10 chars truncated. Fixed 88px width.
- `MESSAGE` — body role (15px), `--text-offwhite`, wraps on narrow viewport; truncates with `…` at single line on 800px kiosk width

**New entries:** push in from top. SSE `log-entry` named event triggers prepend to the displayed list — no page refresh needed (D-29).

**Default filter:** WARN+ (shows WARN and ERROR; hides INFO). User can change to ALL or ERROR only via level dropdown (D-30).

**Entry cap:** 500 entries displayed. "LOAD MORE" button at bottom (label role, amber border) loads next 500 from SQLite API (D-31).

**Settings Logs Tab:** New tab beside NOTIFICATIONS in SettingsPage. Contains: retention days input (default 7, numeric, 1–365 range), "SAVE RETENTION" button. Existing tab pattern and input styles apply.

## New CSS: CRT Scan-Line Sweep (D-37)

Phase 8 adds a second CRT effect — a slow-moving horizontal sweep bar over the existing static scanline grid on `body::after`. Implementation as a separate fixed-position `<div>` with class `.crt-sweep` (not `body::after` which is already occupied by the scanline grid).

```
height: 4px
background: linear-gradient(transparent, rgba(255,255,255,0.025), transparent)
animation: crtSweep 10s linear infinite
pointer-events: none
position: fixed
top: 0; left: 0; right: 0
z-index: 10000
```

```css
@keyframes crtSweep {
  0%   { transform: translateY(-100%); }
  100% { transform: translateY(100vh); }
}
```

Speed: 10s per pass (slow, ambient — not distracting). Subject to `prefers-reduced-motion: reduce` override (existing rule in `globals.css` covers this automatically).

## Interaction Contracts

### Log Viewer Interactions

| Interaction | Behavior |
|-------------|----------|
| Level filter change | Immediate client-side re-filter of loaded entries; no API call |
| Service filter change | Immediate client-side re-filter of loaded entries; no API call |
| EXPORT LOGS | Downloads JSON or CSV of current filtered view. No confirmation modal. Filename: `coruscant-logs-{YYYY-MM-DD}.json` |
| PURGE LOGS | Opens confirmation modal. Modal copy: **"DELETE LOGS"** / "Delete all logs older than {N} days? This cannot be undone." / [CANCEL] [CONFIRM DELETE] buttons. On confirm: POST `/api/logs/purge` — log viewer refreshes to reflect deleted entries |
| LOAD MORE | Appends next 500 entries to bottom of list. Button label updates to "SHOWING {N} ENTRIES — LOAD MORE" |
| Live tail | New SSE `log-entry` events prepend entries to top of list. If user has scrolled down, entries are added silently (no auto-scroll) |

### Settings Logs Tab Interactions

| Interaction | Behavior |
|-------------|----------|
| Retention days input | Numeric input, 1–365, default 7. Inline validation: red border if < 1 or > 365 |
| SAVE RETENTION | POST `/api/settings/logs-retention` with `{ retentionDays: N }`. Success: amber checkmark appears inline for 2s. Error: red "SAVE FAILED" message inline |

### Dashboard: Inline SABnzbd Download (D-10)

When SABnzbd is actively downloading, the SABnzbd tile body shows one additional line below its primary metrics:

```
[filename-truncated...  XX%]     [X.X MB/s]
```

- Left: filename truncated to ~24 chars with `…`, then space, then `XX%` — all `--text-offwhite`, label role (13px)
- Right: download speed in amber, label role (13px)
- This line only renders when `sabnzbd.speed > 0`; hidden when idle

### NowPlayingBanner — Plex Real-Time (D-06/D-07/D-08)

No interaction contract change — existing banner behavior is preserved. The underlying data refresh path changes (Tautulli webhook trigger → immediate PMS re-poll → SSE push) but the frontend component is unchanged. Banner update latency target: < 1s from stream start.
