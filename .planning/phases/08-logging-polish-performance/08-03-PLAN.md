**1. Replace LogsPage.tsx entirely (per D-28 through D-33):**

Props interface:
```typescript
import { useState, useEffect, useRef, useCallback } from 'react'
import type { LogEntry } from '../hooks/useDashboardSSE.js'

interface LogsPageProps {
  lastLogEntry?: LogEntry | null
}
```

**State:**
- `entries: LogEntry[]` — loaded from API, prepended by SSE
- `total: number` — total count from API
- `levelFilter: 'all' | 'warn' | 'error'` — default `'warn'` (D-30: WARN+ default)
- `serviceFilter: string` — default `'all'`
- `loading: boolean`
- `showPurgeModal: boolean`
- `purging: boolean`

**Initial load:** On mount, fetch `GET /api/logs?limit=500&offset=0&level=warn&service=all`. Populate `entries` and `total`.

**Filter changes:** When `levelFilter` or `serviceFilter` changes, re-fetch from API with new params. This is server-side filtering (D-30 says "immediate client-side re-filter" but the data is paginated from SQLite — fetch filtered from server for correct pagination).

**Live tail (D-29):** `useEffect` watching `lastLogEntry` — when it changes and is not null:
- If entry matches current filter (level >= filter level AND service matches), prepend to `entries`
- Cap at 500 entries in state (pop from end if over)
- Only auto-scroll if user is at the top of the list (check `scrollTop < 50`)

**Layout (per UI-SPEC):**
```
[PAGE HEADER]  "SYSTEM LOGS"  (heading role 18px, amber, letter-spacing 0.08em)
[FILTER BAR]   [LEVEL: WARN+] [SERVICE: ALL]   [EXPORT LOGS]  [PURGE LOGS]
[LOG TABLE]    scrollable area
[LOAD MORE (N ENTRIES SHOWN)]
```

**Filter bar:** Flex row with gap 8px. Two `<select>` elements styled with cockpit aesthetic:
- Level select: options `ALL`, `WARN+`, `ERROR ONLY`. Values: `all`, `warn`, `error`.
- Service select: options `ALL` plus unique service names extracted from loaded entries. Values: `all`, service name.
- Both selects: dark background (`var(--space-mid)`), amber border (`var(--border-rest)`), amber text on focus, 44px min height (touch target).

**EXPORT LOGS button** (D-32): Styled button with amber border. On click: trigger `window.location.href = '/api/logs/export?level=${levelFilter}&service=${serviceFilter}&format=json'` to initiate download. No confirmation modal.

**PURGE LOGS button** (D-33): Styled button with red border. On click: set `showPurgeModal = true`.

**Purge modal (D-33):** Fixed overlay, centered modal with dark background. Content:
- Heading: `DELETE LOGS` (heading role, amber)
- Body: `Delete all logs older than {N} days? This cannot be undone.` (body role). N comes from fetched retention value (fetch `/api/settings/logs-retention` on modal open).
- Buttons: `KEEP LOGS` (amber border, dismiss) and `CONFIRM DELETE` (red background, white text).
- On confirm: POST `/api/logs/purge` with `{ olderThanDays: retentionDays }`. On success, re-fetch entries. Close modal.

**Log table:** Rendered as a `<div>` with rows (not HTML `<table>` — easier cockpit styling).

Each row is a flex row with:
- `TIME` — 9 chars `HH:mm:ss` format, label role (12px), `var(--text-offwhite)`. Extract from ISO timestamp: `new Date(entry.timestamp).toLocaleTimeString('en-US', { hour12: false })`.
- `LEVEL` — colored chip: inline-block, label role (12px), uppercase, padding `2px 6px`, border-radius `2px`.
  - INFO: `background: rgba(232,160,32,0.45); color: #E8A020`
  - WARN: `background: rgba(232,160,32,0.8); color: #1a1a1a`
  - ERROR: `background: #FF3B3B; color: #1a1a1a`
  - Fixed width 48px, text-align center.
- `SERVICE` — amber tag: `background: rgba(232,160,32,0.2); border: 1px solid rgba(232,160,32,0.4); color: #E8A020`. Label role, uppercase, padding `2px 6px`, max-width 88px, text-overflow ellipsis.
- `MESSAGE` — body role (15px), `var(--text-offwhite)`, flex: 1, overflow hidden, text-overflow ellipsis, white-space nowrap on 800px kiosk width.

Row spacing: padding `4px 0` per row. Alternating row backgrounds: odd rows get `rgba(232,160,32,0.03)` for subtle zebra striping.

**Empty state:** When no entries match filter:
- Heading: `NO LOGS` (heading role, amber)
- Body: `No log entries match the current filter. Change the level or service filter to broaden the view.`

**LOAD MORE button** (D-31): At bottom of list. Label: `LOAD MORE (${entries.length} ENTRIES SHOWN)`. On click: fetch with offset = entries.length. Append results to existing entries. Hidden when `entries.length >= total`.
