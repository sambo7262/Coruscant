### Task 1 — Fix DOWNLOADS LED color (amber → purple)

In `packages/frontend/src/components/cards/ServiceCard.tsx`, find the `SabnzbdLed` component (around line 335). When `isActive` is true, it currently renders an amber dot. Change to purple:

```tsx
// Before

// After

boxShadow: '0 0 6px var(--cockpit-purple)',
```

`--cockpit-purple` is already defined in the CSS variables. Do not add a new variable.

### Task 2 — Add NETWORK and DOWNLOADS banner strips

Currently `ServiceCard` renders a plain card with an inline amber text label. The MEDIA arr tile has a prominent 20px amber header strip. Apply the same pattern to NETWORK (pihole) and DOWNLOADS (sabnzbd).

In `packages/frontend/src/components/cards/ServiceCard.tsx`, find the `ServiceCard` render function. Currently the card starts with a flex header containing the service name and LED. Replace for pihole and sabnzbd with a banner strip matching the MEDIA tile:

```tsx
// Add at the TOP of the chamfer-card div, before any existing content:
{(service.id === 'pihole' || service.id === 'sabnzbd') && (
  <div style={{
    height: '20px',
    background: 'var(--cockpit-amber)',
    flexShrink: 0,
    display: 'flex',
    alignItems: 'center',
    justifyContent: 'space-between',
    paddingLeft: '6px',
    paddingRight: '8px',
  }}>
    <span style={{
      fontFamily: 'var(--font-mono)',
      fontSize: '9px',
      color: '#1a1a1a',
      letterSpacing: '0.08em',
      fontWeight: 600,
    }}>
      {service.id === 'pihole' ? 'NETWORK' : 'DOWNLOADS'}
    </span>
    {/* Move LED into banner strip */}
    {service.id === 'sabnzbd' && !isUnconfigured
      ? <SabnzbdLed service={service} />
      : <StatusDot status={isUnconfigured ? 'stale' : service.status} />
    }
  </div>
)}
```

When the banner strip is present, remove or hide the duplicate title+LED from the existing inline header so it doesn't appear twice. The existing inline header can remain for other services (non-pihole, non-sabnzbd) that don't have the banner.

### Task 3 — Force DOWNLOADS tile to its own row

In `packages/frontend/src/components/cards/CardGrid.tsx`, the downloads tile currently flows into whatever column is available in the `auto-fit` grid. Force it to start on a new row by wrapping the downloads section in a full-width container:

```tsx
{/* Downloads — always on its own row below media + network */}
{downloadServices.length > 0 && (
  <div style={{ gridColumn: '1 / -1', display: 'grid', gridTemplateColumns: 'repeat(auto-fit, minmax(140px, 1fr))', gap: '8px' }}>
    {downloadServices.map((service) => (
      <ServiceCard key={service.id} service={service} index={globalIndex++} />
    ))}
  </div>
)}
```

This uses `gridColumn: '1 / -1'` to span the full outer grid width, then re-creates the auto-fit grid inside for the download tile(s).

// In the main grid container style:
