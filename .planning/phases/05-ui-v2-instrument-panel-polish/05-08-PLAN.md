<objective>
Add play/pause state indicator to Plex stream display.

The Plex `/status/sessions` API returns each session with a `state` attribute ('playing', 'paused', 'buffering'). This field is not currently extracted by the backend adapter or displayed in the UI.

Add `state` to `PlexStream`, extract it from the Plex API, and show a play/pause icon (icon only, no text label) in StreamRow and the NowPlayingBanner collapsed rail.

Output: Plex streams visually indicate play vs. pause state with an icon.
</objective>

## Tasks

### Task 1 — Add state field to PlexStream

In `packages/shared/src/types.ts`, add the optional `state` field to `PlexStream`:

```ts
export interface PlexStream {
  user: string
  title: string
  deviceName: string
  year?: number
  season?: number
  episode?: number
  progressPercent: number
  quality: string
  transcode: boolean
  mediaType?: 'audio' | 'video'
  albumName?: string
  trackTitle?: string
  state?: 'playing' | 'paused' | 'buffering'  // ADD THIS
}
```

### Task 2 — Extract state from Plex adapter

In `packages/backend/src/adapters/plex.ts`, the sessions response XML is parsed into `PlexMetadataItem` objects. Each session element has a `state` attribute.

Read the plex adapter fully before editing. Find where `PlexMetadataItem` is defined and where session items are mapped to `PlexStream`. Add `state` extraction:

1. Add `state?: string` to `PlexMetadataItem` (or whatever the raw session type is called)
2. In the mapping function, add:
   ```ts
   state: (item.state === 'playing' || item.state === 'paused' || item.state === 'buffering')
     ? item.state
     : undefined,
   ```

Run existing plex adapter tests after the change — they should pass since `state` is optional and test fixtures may not have it (that's fine, `undefined` is valid).

### Task 3 — Play/pause icon in StreamRow

In `packages/frontend/src/components/layout/StreamRow.tsx`, add a play/pause icon alongside the existing content.

Use Unicode or a simple SVG — do NOT add a new icon library dependency. Use Lucide icons if already imported elsewhere in the component, otherwise use Unicode:
- Playing: `▶` (U+25B6) or Lucide `Play`
- Paused: `⏸` (U+23F8) or Lucide `Pause`
- Buffering / undefined: omit icon

Place the icon at the far right of the row, small (10–11px), matching the cockpit-amber color palette. Example:

```tsx
{stream.state && stream.state !== 'buffering' && (
  <span style={{
    fontSize: '10px',
    color: stream.state === 'paused' ? '#666' : 'var(--cockpit-amber)',
    marginLeft: '4px',
    flexShrink: 0,
  }}>
    {stream.state === 'playing' ? '▶' : '⏸'}
  </span>
)}
```

Check what icon imports already exist in StreamRow.tsx before adding new ones.

### Task 4 — Play/pause icon in NowPlayingBanner collapsed rail

Update to include the state icon inline with the title, using the same Unicode/styling approach as Task 3.
