## Tasks

### Task 1 — Fix MEDIA + NETWORK to span full width as two equal columns

**Root cause:** The outer grid uses `repeat(auto-fit, minmax(140px, 1fr))`. At 800px, this creates ~5 columns, so MEDIA and NETWORK each occupy one ~140px column and look like small squares.

**Fix:** In `packages/frontend/src/components/cards/CardGrid.tsx`, change the outer grid's `gridTemplateColumns` from `repeat(auto-fit, minmax(140px, 1fr))` to `repeat(2, 1fr)`. This gives exactly two equal columns: MEDIA (left) and NETWORK (right).

```tsx
// Before
gridTemplateColumns: 'repeat(auto-fit, minmax(140px, 1fr))'

// After
gridTemplateColumns: 'repeat(2, 1fr)'
```

The DOWNLOADS row already uses `gridColumn: '1 / -1'` with its own inner auto-fit grid — leave that unchanged. It will span both columns correctly.

Also update the skeleton placeholder grid to match:
```tsx
gridTemplateColumns: 'repeat(2, 1fr)'
```

### Task 2 — Fix dashboard scroll via body overflow useEffect

**Root cause:** `overflowY: hidden` on `<main>` doesn't prevent `<body>` from scrolling. The `alignItems: 'stretch'` change made the grid taller than 480px on some viewports, causing body scroll to reappear.

**Fix:** In `packages/frontend/src/App.tsx`, add a `useEffect` that locks `document.body.style.overflow` when on the dashboard route and restores it on leave:

```tsx
import { useEffect } from 'react'

// Inside App(), after const isDashboard = location.pathname === '/'
useEffect(() => {
  document.body.style.overflow = isDashboard ? 'hidden' : ''
  return () => {
    document.body.style.overflow = ''
  }
}, [isDashboard])
```

Also remove the `overflowY: isDashboard ? 'hidden' : 'auto'` from the `<main>` style — the body-level lock is the correct approach and the main override is redundant:
```tsx
// Remove overflowY from main style — body lock handles it
<main style={{ position: 'relative', zIndex: 1, paddingTop: '128px', paddingBottom: '40px' }}>
```
