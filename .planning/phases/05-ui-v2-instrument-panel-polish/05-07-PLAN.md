### Task 4 — Reactive LEDs

In `packages/frontend/src/components/cards/ServiceCard.tsx`, update LED color logic:

**SABnzbd (DOWNLOADS) LED:**
- Active (amber) when: `service.status === 'online'` AND `metrics?.speed > 0` OR `metrics?.queueCount > 0`
- Online/idle (green): online but no active download
- Offline/warning/stale: existing colors

```tsx
function getLedStyle(service: ServiceStatus): React.CSSProperties {
  if (service.status === 'offline') return { background: '#FF3B3B' }
  if (service.status === 'warning') return { background: '#FF8C00' }
  if (service.status === 'stale')   return { background: '#666' }
  // online — check for active state
  if (service.id === 'sabnzbd') {
    const m = service.metrics as SabnzbdMetrics | undefined
    if (m && (m.speedMBs > 0 || m.queueCount > 0)) {
      return { background: 'var(--cockpit-amber)', boxShadow: '0 0 6px var(--cockpit-amber)' }
    }
  }
  // arr services — active when queue > 0
  if (['radarr','sonarr','lidarr','prowlarr','bazarr','readarr'].includes(service.id)) {
    const m = service.metrics as ArrMetrics | undefined
    if (m && m.queueCount > 0) {
      return { background: 'var(--cockpit-amber)', boxShadow: '0 0 6px var(--cockpit-amber)' }
    }
  }
  return { background: 'var(--cockpit-green, #4ADE80)' }
}
```

Note: Check the actual type/shape of arr metrics in `packages/shared/src/types.ts` before accessing `queueCount`. Only add the arr queue LED if the data is already available in metrics. If not, leave arr LEDs as green-online and note the gap.
