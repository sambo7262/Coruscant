## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, LED badge padding, dot-leader fill |
| sm | 8px | Intra-card padding, label-to-value gaps |
| md | 16px | Card internal padding, settings field spacing |
| lg | 24px | Section gaps within detail views |
| xl | 32px | Between header strip and card grid |
| 2xl | 48px | Major layout breaks (header zone, grid zone, rail zone) |
| 3xl | 64px | Reserved — not used in Phase 4 |

Exceptions:
- Touch targets for the AppHeader strip tap zone and NowPlayingBanner tap zone: minimum 44px height (mobile touch compliance — viewport is 800x480 physical Pi touchscreen)
- NAS expanded panel row height: 28px (compact data density; tighter than md to fit per-disk rows without scroll)
- Plex stream rows in expanded rail: 32px per row

## Color

Source: `packages/frontend/src/styles/globals.css` and CONTEXT.md (Phase 2 D-01 through D-07, Phase 4 D-05).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#080C14` (var(--space-deep)) | Full viewport background, page surface |
| Secondary (30%) | `#0D0D0D` (var(--bg-panel)) + `#0E1520` (var(--space-mid)) | Card faces, header strip, rail surface, expanded panel background |
| Accent (10%) | `#E8A020` (var(--cockpit-amber)) | See reserved-for list below |
| Healthy/Online | `#4ADE80` (var(--cockpit-green)) | Pi-hole blocking-active LED, Plex playing LED, NAS healthy LED |
| Warning/Degraded | `#E8A020` (var(--cockpit-amber)) | Pi-hole blocking-disabled LED, NAS stale-data state, image update LED |
| Down/Critical | `#FF3B3B` (var(--cockpit-red)) | Pi-hole offline LED, NAS unreachable LED |
| Unconfigured | `#666666` (var(--cockpit-grey)) | Services not yet configured in Settings |
| Destructive | `#FF3B3B` (var(--cockpit-red)) | Destructive confirmation text only (no destructive actions in Phase 4) |

Accent (`#E8A020`) reserved for:
- Seam lines and chamfer borders at rest
- Section labels in CardGrid ("RICH DATA", "ACTIVITY", "STATUS")
- Metric value text on AppHeader strip
- Dot-leader fill characters in detail views
- Animated border traces on service cards
- NowPlayingBanner rail border and label "NOW PLAYING"
- Image update LED blink when updates are available

**LED semantics (from Phase 3, locked, applied to Phase 4 new services):**

| State | LED Color | Applies To |
|-------|-----------|------------|
| Healthy / blocking active | Green `#4ADE80` | Pi-hole (blocking on and reachable) |
| Warning / blocking disabled | Amber `#E8A020` | Pi-hole (reachable but blocking disabled) |
| Down / unreachable | Red `#FF3B3B` | Pi-hole (offline), NAS (unreachable) |
| Unconfigured | Grey `#666666` | Pi-hole / NAS / Plex before Settings are saved |
| Stale | Grey `#666666` | NAS header strip when DSM session has expired |

**NAS temperature color ramp (already in AppHeader.tsx):**

| Threshold | Color |
|-----------|-------|
| < 95°F (< 35°C) | Amber `#E8A020` — normal |
| 95–113°F (35–45°C) | Orange `#FF8C00` — warm |
| ≥ 114°F (≥ 45.5°C) | Red `#FF3B3B` — critical |

## Layout Contract

Source: CONTEXT.md layout decisions (locked) + additional_context from orchestrator.

### Viewport Target

Primary: 800×480px (Raspberry Pi kiosk touchscreen). Secondary: mobile phone portrait. No horizontal or vertical scroll on the main dashboard — everything fits one viewport.

### Zone Allocation (top to bottom, fixed height budget)

```
┌─────────────────────────────────────────────────────────┐
│  AppHeader — fixed top, ~48px collapsed                 │
│  [NAS stats strip: CPU% RAM% ↑Mbps ↓Mbps Disk% Temp°C] │
│  [Tap → expands downward, overlaps card zone]           │
├─────────────────────────────────────────────────────────┤
│  Card Grid — fills remaining viewport minus rail        │
│  Two-column header area:                                │
│    LEFT COLUMN: media arr stack                         │
│      Sonarr | Radarr                                    │
│      Lidarr | Prowlarr                                  │
│      Readarr | (empty or SABnzbd)                       │
│    RIGHT COLUMN: Pi-hole rich card                      │
│  Below header area: SABnzbd download status             │
│  (exact card arrangement finalized by executor based    │
│   on live 800×480 fit testing)                          │
├─────────────────────────────────────────────────────────┤
│  NowPlayingBanner — fixed bottom, ~40px collapsed       │
│  [Streams ticker — tap → expands upward]                │
└─────────────────────────────────────────────────────────┘
```

### AppHeader NAS Strip (collapsed)

- Fixed top, full width
- Height: 48px minimum (touch target compliant)
- Layout: 3-column CSS grid (logo/nav | NAS gauges | action icons) — existing pattern from Phase 2
- NAS gauges: `CPU% · RAM% · ↑0.0 ↓0.0 Mbps · Disk% · 38°C` — space-separated values at 12px label + 14px value
- Stale state: all gauge values replaced by "—" in grey `#666666`; StaleIndicator component appears
- Unconfigured state: NAS gauges zone shows "NAS NOT CONFIGURED" in grey at 12px
- Tap anywhere on the strip (except nav icons) → expand panel downward

### AppHeader NAS Panel (expanded, downward drawer)

- Framer Motion: `height: 0 → auto`, `opacity: 0 → 1`, duration 0.3s, ease `easeInOut`
- Expands downward from header, overlaps card grid (z-index above cards, below CRT overlay)
- Background: `var(--space-mid)` `#0E1520` with 1px amber bottom border
- Conditional sections (D-19): only rendered when DSM returns data for that section
  - Per-disk rows: disk name · read MB/s · write MB/s · temp (colored by ramp above)
  - Docker daemon stats: CPU% · RAM% · network
  - Fan speeds: one row per fan (only on NAS models that return fan data)
  - Image update LED: amber blinking LED + "UPDATES AVAILABLE" label if any images have updates; grey static LED + "UP TO DATE" if none
- Tap strip again or tap outside → collapse

### NowPlayingBanner (rail — bottom)

- Fixed bottom, full width
- Collapsed height: 40px
- Collapsed state shows: `● N STREAM(S) ACTIVE` + scrolling ticker of first stream title · user
- Ticker scroll: CSS `@keyframes marquee` horizontal translate; only activates when text overflows rail width (existing `shouldScroll` logic)
- When no active streams: dim "NO ACTIVE STREAMS" label at 12px grey — rail stays visible at 40px (D-11: idle state visible, not hidden)
- Expanded state (tap to open, expands upward):
  - One `StreamRow` per active stream: USER › TITLE left, QUAL/DIRECT right, 1px amber progress line below
  - Plex server stats section below streams: BANDWIDTH · PLEX CPU% · PLEX RAM%
  - Backdrop overlay behind expanded rail (existing pattern)
  - Tap backdrop or tap rail again → collapse

### Pi-hole Card (grid)

- Standard service card (chamfered instrument panel, same as arr cards)
- Instrument body type: stats readout (not dot matrix, not gauge columns)
- Card face shows 4 metrics in a 2×2 grid:
  - Top-left: QPM (queries per minute) value + "QPM" label
  - Top-right: LOAD (system load float) value + "LOAD" label
  - Bottom-left: MEM% (memory usage) value + "MEM%" label
  - Bottom-right: STATUS LED ("BLOCKING" or "DISABLED") text label
- LED: top-right of card header per existing ServiceCard pattern (green/amber/red/grey)
- Service name label: "PI-HOLE" in heading style

## Interaction Contract

### Tap Targets

| Element | Min Touch Target | Behavior |
|---------|-----------------|----------|
| AppHeader NAS strip | 44px height | Expand/collapse NAS panel |
| NowPlayingBanner rail | 44px height | Expand/collapse Plex rail |
| Pi-hole card | Full card area | Navigate to Pi-hole detail view |
| Arr service cards | Full card area | Navigate to service detail view (unchanged) |
| NAS panel backdrop (when open) | Full viewport | Collapse NAS panel |
| Plex rail backdrop (when open) | Full viewport minus rail | Collapse Plex rail |
| Settings tab items | 44px height | Open that service's settings tab |

### Animation Timing

| Animation | Duration | Easing | Property |
|-----------|----------|--------|----------|
| NAS panel expand/collapse | 300ms | easeInOut | height + opacity |
| Plex rail expand/collapse | 300ms | easeInOut | height + opacity (existing) |
| Pi-hole card border trace | 2000ms | linear | stroke-dashoffset (existing pattern) |
| LED breathe (healthy) | 2400ms | ease-in-out | opacity (existing ledBreathe keyframe) |
| LED pulse (warning) | 800ms | ease-in-out | opacity (existing ledPulseWarn keyframe) |
| LED flash (down) | 400ms | ease-in-out | opacity (existing ledFlashDown keyframe) |
| Image update LED blink | 1200ms | ease-in-out | opacity (same as ledPulseWarn) |
| Ticker scroll (Plex rail) | Auto (CSS marquee) | linear | translateX |
| NAS temp bar fill | 600ms | ease | height (existing VerticalBar transition) |

All animations use `transform` and `opacity` only (DASH-08 requirement).

### Settings Page Additions

Three new tabs: PI-HOLE · PLEX · NAS

| Tab | Fields | Pattern |
|-----|--------|---------|
| PI-HOLE | URL (text), Password (masked + eye-toggle) | 2-field — same as Phase 3 arr tabs |
| PLEX | URL (text), Plex Token (masked + eye-toggle, labeled "Plex Token" not "API Key") | 2-field |
| NAS | URL (text), DSM Username (text), DSM Password (masked + eye-toggle) | 3-field — unique exception |

Settings notes (rendered below the form, 12px grey italic):
- PI-HOLE tab: "Pi-hole v6 or higher required."
- NAS tab: "Requires an admin-level DSM account."

TEST button behavior (all three tabs):
- Pi-hole: POST `/api/auth` with password → success shows version, failure shows error message
- Plex: GET `/` with `X-Plex-Token` header → success shows server name, failure shows error
- NAS: SYNO.API.Auth login → success shows DSM version, failure shows error

Inline result display: same cockpit pill pattern as Phase 3 (green "CONNECTED · version" or red "ERROR · message").

## Polling Intervals

| Service | Interval | Rationale |
|---------|----------|-----------|
| Pi-hole | 30 seconds | DNS stats; real-time not required; same cadence as arr services |
| Plex | 10 seconds | Stream progress needs to be current for Now Playing ticker accuracy |
| NAS (hardware stats) | 10 seconds | Live CPU/RAM/temp meaningful at this cadence; DSM session valid ~30 min |
| NAS (image update check) | 2× per day (43200 seconds) | Separate timer; not part of main poll cycle; result cached in snapshot |
| Arr services (Sonarr, Radarr, etc.) | 30 seconds | Unchanged from Phase 3 |
| SABnzbd | 10 seconds | Unchanged from Phase 3 |
