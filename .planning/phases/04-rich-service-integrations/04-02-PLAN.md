**packages/backend/src/adapters/pihole.ts** — Create Pi-hole v6 adapter:

1. Create a `PiholeSession` class (module-level singleton Map keyed by baseUrl):
   - `ensureSession(baseUrl, password)`: if cached sid is valid (Date.now() < validUntil - 10_000), return it; otherwise POST to `${baseUrl}/api/auth` with `{ password }`, extract `session.sid` and `session.validity`, store and return sid.
   - `invalidate()`: clear sid and validUntil.
   - On 401 from any subsequent call, invalidate and retry once.

2. Export `async function pollPihole(baseUrl: string, password: string): Promise<ServiceStatus>`:
   - Call `ensureSession(baseUrl, password)` to get sid.
   - Make 3 parallel requests with header `X-FTL-SID: {sid}`:
     - `GET ${baseUrl}/api/stats/summary` — extract `queries.total`, `queries.blocked`, `queries.percent_blocked`, `queries.frequency`, `gravity.domains_being_blocked`
     - `GET ${baseUrl}/api/dns/blocking` — extract `blocking` field ("enabled" or "disabled")
     - `GET ${baseUrl}/api/info/system` — extract `system.cpu.load[0]` (1min load), memory percent = `(system.memory.ram.used / system.memory.ram.total) * 100`
   - Use `Promise.all()` for the 3 parallel requests, wrapped in a single try/catch.
   - If any request returns 401, call `session.invalidate()`, retry the entire poll once.
   - Map blocking state: `blocking === 'enabled'` → status `'online'` (green LED), `blocking === 'disabled'` → status `'warning'` (amber LED per D-05).
   - Return `ServiceStatus` with `id: 'pihole'`, `name: 'Pi-hole'`, `tier: 'rich'`, `configured: true`, and `metrics`:
     ```typescript
     metrics: {
       blockingActive: blocking === 'enabled',
       queriesPerMinute: queries.frequency ?? 0,
       load1m: system.cpu.load?.[0] ?? 0,
       memPercent: memUsedPercent,
       totalQueriesDay: queries.total ?? 0,
       totalBlockedDay: queries.blocked ?? 0,
       percentBlocked: queries.percent_blocked ?? 0,
       domainsBlocked: gravity.domains_being_blocked ?? 0,
     }
     ```
   - On catch: return `status: 'offline'`, `configured: true`, `metrics: { error: reason }`.

**packages/backend/src/adapters/plex.ts** — Create Plex adapter:

1. Export `async function pollPlex(baseUrl: string, token: string): Promise<{ streams: PlexStream[]; serverStats?: PlexServerStats }>`:
   - Make 2 parallel requests, both with headers `{ 'X-Plex-Token': token, 'Accept': 'application/json' }`:
     - `GET ${baseUrl}/status/sessions` — map `MediaContainer.Metadata[]` to `PlexStream[]`
     - `GET ${baseUrl}/statistics/resources?timespan=6` — map most recent `StatisticsResources` entry to `PlexServerStats`
   - Stream mapping from each Metadata item:
     - `user`: `item.User?.title ?? 'Unknown'`
     - `title`: `item.grandparentTitle ? \`${item.grandparentTitle}\` : item.title` (show name for episodes)
     - `season`: `item.parentIndex ?? undefined`
     - `episode`: `item.index ?? undefined`
     - `year`: `item.year ?? undefined`
     - `progressPercent`: `Math.round((item.viewOffset / item.duration) * 100)` (NOT TranscodeSession.progress)
     - `quality`: `item.Media?.[0]?.videoResolution ? \`${item.Media[0].videoResolution}p\` : 'unknown'`
     - `transcode`: `item.TranscodeSession?.videoDecision === 'transcode'`
   - PlexServerStats mapping from most recent StatisticsResources entry:
     - `processCpuPercent`: `entry.processCpuUtilization ?? 0`
     - `processRamPercent`: `entry.processMemoryUtilization ?? 0`
     - `bandwidthMbps`: sum of all `stream.Session?.bandwidth` values / 1000 (kbps to Mbps)
   - On error: return `{ streams: [], serverStats: undefined }`.

2. Note: Plex does NOT produce a ServiceStatus entry. It only provides streams and serverStats to the snapshot. The PollManager stores these separately.

**packages/backend/src/adapters/nas.ts** — Create NAS adapter:

1. Create a `DsmSession` class (module-level singleton Map keyed by baseUrl):
   - `ensureSession(baseUrl, username, password)`: if cached sid valid, return it; otherwise GET `${baseUrl}/webapi/entry.cgi` with params `{ api: 'SYNO.API.Auth', version: 6, method: 'login', account: username, passwd: password, format: 'sid' }`. Extract `data.sid`. DSM sessions last ~30 minutes; set `validUntil = Date.now() + 25 * 60 * 1000` (25 min, conservative).
   - `invalidate()`: clear sid.
   - On error code 119 from any subsequent call, invalidate and retry once.

2. Export `async function pollNas(baseUrl: string, username: string, password: string): Promise<NasStatus>`:
   - Call `ensureSession()` to get sid.
   - Make 3 parallel requests with `_sid={sid}` as query parameter:
     - `GET /webapi/entry.cgi?api=SYNO.Core.System.Utilization&version=1&method=get&type=current` — extract cpu (user_load + system_load + other_load), memory.real_usage, network[0].rx and .tx
     - `GET /webapi/entry.cgi?api=SYNO.Core.System&version=1&method=info&type=storage` — extract hdd_info[].temp, vol_info[].used_size/total_size, top-level temperature
     - `GET /webapi/entry.cgi?api=SYNO.Core.Hardware.FanSpeed&version=1&method=get` — extract fan_speed[] array
   - Handle DSM error code 119 (invalid session): invalidate session, retry once.
   - For each request, if DSM returns `success: false`, treat that metric as unavailable (don't crash the whole poll).
   - Map results to NasStatus:
     - `cpu`: `data.cpu.user_load + data.cpu.system_load + (data.cpu.other_load ?? 0)`
     - `ram`: `data.memory.real_usage` (already a percent)
     - `networkMbpsUp`: `(network[0]?.tx ?? 0) * 8 / 1_000_000` (bytes/sec to Mbps)
     - `networkMbpsDown`: `(network[0]?.rx ?? 0) * 8 / 1_000_000`
     - `cpuTempC`: `storageData.temperature` (optional)
     - `volumes`: map `vol_info[]` → `{ name, usedPercent: parseInt(used_size)/parseInt(total_size)*100 }`
     - `disks`: map `hdd_info[]` → `{ id, name, tempC: item.temp }` (only if hdd_info exists and has items)
     - `fans`: map `fan_speed[]` → `{ id, rpm }` (set `undefined` if empty array or request failed — NOT empty array, per D-19)
   - On catch: return default NasStatus with `cpu: 0, ram: 0, networkMbpsUp: 0, networkMbpsDown: 0, volumes: []`.

3. Export `async function checkNasImageUpdates(baseUrl: string, username: string, password: string): Promise<boolean>`:
   - Call `ensureSession()` to get sid.
   - `GET /webapi/entry.cgi?api=SYNO.Docker.Image&version=1&method=list&_sid={sid}`
   - Return `data.images?.some(img => img.is_update_available === true) ?? false`
   - On error: return `false` (defensive, per RESEARCH.md open question #1).
