    - Fields:

      - **API Key field (per D-08, D-09):** `<input type="password" />` by default. Eye icon button (unicode `\u{1F441}` or a simple SVG eye) positioned at right edge of input. Clicking toggles between `type="password"` and `type="text"`. Same styling as URL field. Placeholder: `"API Key"`
      - **TEST button (per D-11, D-12):** Styled as cockpit momentary button — `background: rgba(232,160,32,0.15)`, `border: 1px solid var(--cockpit-amber)`, `color: var(--cockpit-amber)`, `padding: 8px 20px`, uppercase text `TEST`. While in-flight: text changes to `TESTING...`, button is `disabled`.
      - **Inline test result (per D-13, D-14):**
        - Below the TEST button, a result line appears ONLY after a test completes
        - Success: green 10px circle + `CONNECTED  v3 - N warnings` text in green
        - Failure: red 10px circle + `FAILED: [message]` text in red
        - Result clears (disappears) when user edits either the URL or API Key field (per D-14)
      - **SAVE button (per D-05):** Same cockpit style as TEST but with filled amber background: `background: var(--cockpit-amber)`, `color: #0D0D0D`, `font-weight: 600`. Text: `SAVE`. Per-tab save — only saves the current service's config.

    **State management:**
    - `useState` for: `activeTab`, `url`, `apiKey`, `showKey` (boolean), `testResult` (null | {success, message}), `testing` (boolean), `saving` (boolean)
    - On tab change: fetch `GET /api/settings/{newTabId}` and populate `url` from response `baseUrl`, clear `apiKey` (never returned by backend), clear `testResult`. If `hasApiKey` is true, show a placeholder indicator like `"********"` but the actual input value is empty — user must re-enter to change.
    - On mount: fetch current tab's config

    **SAVE handler:**
    - POST to `/api/settings/${activeTab}` with `{ baseUrl: url, apiKey: apiKey }`
    - On success: briefly show a "Saved" confirmation (optional, 2-second timeout), then the tab LED will update on next SSE push (per D-06)
    - If apiKey is empty string and url is empty string, this effectively disables the service

    **TEST handler:**
    - Set `testing = true`, disable button
    - POST to `/api/test-connection/${activeTab}` with `{ baseUrl: url, apiKey: apiKey }`
    - On response: set `testResult = response`, `testing = false`
    - If fetch itself fails (network): set `testResult = { success: false, message: 'Network error' }`
