    - GET /api/settings/radarr returns {serviceName: "radarr", baseUrl: "http://localhost:7878", hasApiKey: true, enabled: true} — NEVER returns plaintext apiKey
    - GET /api/settings/nonexistent returns {serviceName: "nonexistent", baseUrl: "", hasApiKey: false, enabled: false}

    Test connection routes:
    - POST /api/test-connection/radarr with body {baseUrl, apiKey} calls the service and returns {success: true, message: "Connected v3 - 0 warnings"} or {success: false, message: "ECONNREFUSED"}
    - POST /api/test-connection/sabnzbd uses SABnzbd queue endpoint instead of arr health

  <action>
    1. Create `packages/backend/src/routes/settings.ts`:
       - Export `async function settingsRoutes(fastify: FastifyInstance)` following the same registration pattern as healthRoutes.
       - Define valid service IDs as a const array: `['radarr', 'sonarr', 'lidarr', 'bazarr', 'prowlarr', 'readarr', 'sabnzbd']`

       **POST /api/settings/:serviceId** — Upsert service config:
       - Validate serviceId is in the valid list, return 400 if not
       - Read `ENCRYPTION_KEY_SEED` from `process.env.ENCRYPTION_KEY_SEED` (fail with 500 if missing)
       - If `body.apiKey` is non-empty string: encrypt with `encrypt(body.apiKey, seed)` and store
       - If `body.apiKey` is empty string AND `body.baseUrl` is empty: set `enabled: false`, clear encryptedApiKey
       - Otherwise: upsert row in serviceConfig table using Drizzle `db.insert(...).values({...}).onConflictDoUpdate({ target: serviceConfig.serviceName, set: { baseUrl, encryptedApiKey, enabled: true, updatedAt: new Date().toISOString() } })`
       - After DB write: call `pollManager.reload(serviceId, { baseUrl: body.baseUrl, apiKey: body.apiKey })` to hot-reload polling (per D-06)
       - Return `{ ok: true }`

       **GET /api/settings/:serviceId** — Read config (safe):
       - Query serviceConfig by serviceName
       - If not found: return `{ serviceName: serviceId, baseUrl: '', hasApiKey: false, enabled: false }`
       - If found: return `{ serviceName: row.serviceName, baseUrl: row.baseUrl, hasApiKey: row.encryptedApiKey !== '', enabled: row.enabled }` — NEVER return encryptedApiKey or decrypted key

       **GET /api/settings** — List all services config status:
       - Return array of `{ serviceName, baseUrl, hasApiKey, enabled }` for all 7 services (filling in defaults for unconfigured ones)

    2. Create `packages/backend/src/routes/test-connection.ts`:
       - Export `async function testConnectionRoutes(fastify: FastifyInstance)`

       **POST /api/test-connection/:serviceId**:
       - Accepts body `{ baseUrl: string, apiKey: string }`
       - Validates serviceId is in the valid list
       - For arr services (radarr/sonarr/lidarr/prowlarr/readarr): make GET to `${baseUrl}/api/v3/health` with `X-Api-Key` header, 10-second timeout
         - On 2xx: parse health array, return `{ success: true, message: "Connected v3 - N warnings" }` where N = count of Warning/Error items
         - On error: return `{ success: false, message: error.message }` (e.g., "timeout", "ECONNREFUSED", "Request failed with status code 401")
       - For bazarr: make GET to `${baseUrl}/api/system/status?apikey=${apiKey}`, 10-second timeout
         - On 2xx: return `{ success: true, message: "Connected" }`
         - On error: return `{ success: false, message: error.message }`
       - For sabnzbd: make GET to `${baseUrl}/api?mode=queue&output=json&apikey=${apiKey}`, 10-second timeout
         - On 2xx with valid queue object: return `{ success: true, message: "Connected - queue has N items" }`
         - On error: return `{ success: false, message: error.message }`
       - ALWAYS return HTTP 200 — the success/failure is in the response body, not HTTP status. This makes frontend handling simpler.

    3. Update `packages/backend/src/index.ts`:
       - Import `settingsRoutes` from `'./routes/settings.js'`
       - Import `testConnectionRoutes` from `'./routes/test-connection.js'`
       - Register both after existing route registrations:
         ```typescript
         await fastify.register(settingsRoutes)
         await fastify.register(testConnectionRoutes)
         ```
       - After database initialization block, add PollManager startup:
         ```typescript
         import { pollManager } from './poll-manager.js'
         import { decrypt } from './crypto.js'
         // ... after initDb and probe ...
         // Load saved service configs and start polling
         try {
           const db = getDb()
           const configs = db.select().from(serviceConfig).all()
           const seed = process.env.ENCRYPTION_KEY_SEED ?? ''
           for (const cfg of configs) {
             if (cfg.enabled && cfg.baseUrl && cfg.encryptedApiKey) {
               const apiKey = decrypt(cfg.encryptedApiKey, seed)
               await pollManager.reload(cfg.serviceName, { baseUrl: cfg.baseUrl, apiKey })
             }
           }
         } catch (err) {
           fastify.log.warn({ err }, 'Failed to load service configs for polling')
         }
         ```
       - Import `serviceConfig` from `'./schema.js'`
