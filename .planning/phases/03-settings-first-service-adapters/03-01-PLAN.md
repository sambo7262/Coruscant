  <behavior>
    Arr adapter:
    - Given a 200 response with empty array from /api/v3/health, returns ServiceStatus with status 'online'
    - Given a 200 response with items having type 'Warning' or 'Error', returns status 'warning'
    - Given a 200 response with only type 'Ok' or 'Notice' items, returns status 'online'
    - Given a network error (ECONNREFUSED), returns status 'offline'
    - Given a 401 response, returns status 'offline' (per D-19)

    SABnzbd adapter:
    - Given a queue response with status 'Downloading' and kbpersec '10240', returns speedMBs=10, status 'online'
    - Given a slot with status 'Failed', returns hasFailedItems=true, service status 'warning'
    - Given status 'Downloading' with noofslots > 0, returns service status 'online' (purple is a frontend concern)
    - Given a network error, returns status 'offline'

    PollManager:
    - getSnapshot() returns a DashboardSnapshot with all services represented
    - reload(serviceId, config) starts polling for that service
    - reload(serviceId, null) marks service as unconfigured with configured=false
    - stopAll() clears all timers
  </behavior>
  <action>
    1. Create `packages/backend/src/adapters/arr.ts`:
       - Export `async function pollArr(serviceId: string, serviceName: string, baseUrl: string, apiKey: string): Promise<ServiceStatus>`
       - Makes GET request to `${baseUrl}/api/v3/health` with header `X-Api-Key: ${apiKey}` and 10-second timeout via axios
       - Status mapping:
         - Network error / timeout / non-2xx → `{ status: 'offline', configured: true }`
         - 200 + empty array → `{ status: 'online', configured: true }`
         - 200 + any item with `type === 'Warning' || type === 'Error'` → `{ status: 'warning', configured: true, metrics: { healthWarnings: items.filter(i => i.type === 'Warning' || i.type === 'Error') } }`
         - 200 + only Ok/Notice → `{ status: 'online', configured: true }`
       - All arr services (Radarr, Sonarr, Lidarr, Prowlarr, Readarr) use this same function. Bazarr does NOT.
       - Set `tier: 'status'`, `lastPollAt: new Date().toISOString()`

    2. Create `packages/backend/src/adapters/bazarr.ts`:
       - Export `async function pollBazarr(baseUrl: string, apiKey: string): Promise<ServiceStatus>`
       - Makes GET request to `${baseUrl}/api/system/status?apikey=${apiKey}` with 10-second timeout
       - Status mapping: 2xx → `{ id: 'bazarr', name: 'Bazarr', tier: 'status', status: 'online', configured: true }`. Any error → `{ status: 'offline', configured: true }`.
       - Bazarr is ALWAYS green or red, never amber (no health-warning equivalent).

    3. Create `packages/backend/src/adapters/sabnzbd.ts`:
       - Export `async function pollSabnzbd(baseUrl: string, apiKey: string): Promise<ServiceStatus>`
       - Makes GET request to `${baseUrl}/api?mode=queue&output=json&apikey=${apiKey}` with 10-second timeout
       - Parse response.data.queue:
         - `speedMBs = parseFloat(queue.kbpersec) / 1024` (kbpersec is a string)
         - `queueCount = queue.noofslots` (integer)
         - `progressPercent` = first active slot's percentage if present, else 0
         - `hasFailedItems = queue.slots?.some(s => s.status === 'Failed') ?? false`
         - `sabStatus = queue.status` (raw string)
       - Status mapping:
         - Network error → `{ status: 'offline', configured: true }`
         - `hasFailedItems === true` → `{ status: 'warning', configured: true }` (amber per SVCACT-02)
         - Otherwise → `{ status: 'online', configured: true }`
       - Set `tier: 'activity'`, metrics populated as `SabnzbdMetrics` shape.

    4. Create `packages/backend/src/poll-manager.ts`:
       - Export `class PollManager`:
         - `private timers: Map<string, NodeJS.Timeout>`
         - `private state: Map<string, ServiceStatus>` — holds latest status per service
         - Constructor accepts no args. Initializes all 8 services (radarr, sonarr, lidarr, bazarr, prowlarr, readarr, sabnzbd + NAS/Plex/Pi-hole stubs) with unconfigured status: `{ status: 'stale', configured: false }`
         - `async reload(serviceId: string, config: { baseUrl: string; apiKey: string } | null)`:
           - Clear existing timer for serviceId if any
           - If config is null or empty → set unconfigured status, return
           - Otherwise: run immediate first poll, then set interval
           - Poll intervals: arr services = 45_000ms, sabnzbd = 10_000ms (per D-20)
           - Adapter selection: radarr/sonarr/lidarr/prowlarr/readarr → pollArr; bazarr → pollBazarr; sabnzbd → pollSabnzbd
         - `getSnapshot(): DashboardSnapshot`:
           - Returns `{ services: [...this.state.values()], nas: MOCK_NAS, streams: MOCK_STREAMS, timestamp: new Date().toISOString() }`
           - NAS, Plex, Pi-hole data remain mock for Phase 3 (only arr + sabnzbd are live). Import mock NAS/streams from existing generator or inline stub.
         - `stopAll()` — clears all timers (for graceful shutdown and tests)
       - Export a singleton instance: `export const pollManager = new PollManager()`

    5. Create test files:
       - `packages/backend/src/__tests__/arr-adapter.test.ts`: Mock axios to test all status mappings. Use `vi.mock('axios')`.
       - `packages/backend/src/__tests__/sabnzbd-adapter.test.ts`: Mock axios to test queue parsing, failed items detection, speed calculation.
