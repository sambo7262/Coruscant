# Phase 2 — UI Design Contract

> Visual and interaction contract for the Coruscant core UI shell. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | CONTEXT.md — no component library installed; Phase 2 builds from scratch |
| Preset | not applicable | shadcn not initialized; Tron/terminal aesthetic requires bespoke dark system |
| Component library | none (custom components) | RESEARCH.md — handbuilt `components/ui/`, `components/cards/`, `components/layout/` |
| Icon library | lucide-react 1.7.0 | RESEARCH.md — recommended over heroicons; tree-shakeable, actively maintained |
| Font | `JetBrains Mono` loaded via `@font-face`; fallback: `ui-monospace, SFMono-Regular, Menlo, monospace` | CONTEXT.md D-12 — monospace throughout; system stack first |

**shadcn Gate Decision:** shadcn was intentionally skipped. The Tron/Grid aesthetic requires a bespoke dark design system (`background: #0a0a0f`, glowing borders, CSS animation keyframes) that is incompatible with a neutral shadcn preset. Building custom components is the correct choice for this phase and was confirmed by CONTEXT.md code context (App.tsx already sets inline Tron colors without a component library).

**Registry Safety Gate:** Not applicable — no shadcn, no third-party component registries in scope.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, status dot inline padding |
| sm | 8px | Card internal padding (icon row), section label vertical margin |
| md | 16px | Card padding, header strip padding, default element spacing |
| card-inner | 12px | Service card internal padding (between sm and md — 12 = 3×4, justified by card height constraint of 88px) |
| lg | 24px | Grid gap between cards, section spacing |
| xl | 32px | Header total height target, page horizontal padding on mobile |
| 2xl | 48px | Major layout breaks (e.g. space between card grid and top of Now Playing banner at rest) |
| 3xl | 64px | Page-level bottom padding to clear fixed Now Playing banner |

Exceptions:
- **Touch targets:** Icon buttons in header (Settings ⚙, Logs ≡) implement the iOS HIG 44px minimum via `width: 44px; height: 44px; display: flex; align-items: center; justify-content: center` around a 24px icon. No non-multiple-of-4 padding value is used.
- **Now Playing banner collapsed height:** 48px (fits one line of text with comfortable touch target for tap-to-expand).
- **Now Playing banner expanded height:** `min(60vh, 320px)` — caps at 320px on tall phones; never covers more than 60% of viewport.
- **Status dot size:** 8px diameter (not on the spacing scale — purely visual).

## Typography

All text uses the monospace font stack declared in Design System above. No serif or sans-serif fallback anywhere in the UI.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 20px | 700 (Bold) | 1.2 | "CORUSCANT" app title in header |
| Heading | 16px | 700 (Bold) | 1.2 | Section labels (STATUS, ACTIVITY, RICH DATA, SMART HOME), service detail page title, Now Playing expanded stream titles |
| Body | 14px | 400 (Regular) | 1.5 | Card service name, NAS stats strip values, banner ticker text, detail view metric labels and values |
| Label | 12px | 400 (Regular) | 1.4 | Stale-data indicator badge, stream quality string, transcode/direct-play label, settings stub placeholder text |

**Weights in use: 2 — 400 (Regular) for Body and Label roles; 700 (Bold) for Display and Heading roles.**

**Letter-spacing:** Apply `letter-spacing: 0.08em` to Display and Heading roles only — reinforces terminal aesthetic without making body text hard to read.

**Text transform:** Section labels (STATUS, ACTIVITY, etc.) are uppercase in the data source (`DashboardSnapshot` tier values are lowercase: `'status' | 'activity' | 'rich'`). Apply `text-transform: uppercase` via CSS, never hardcode uppercase in strings.

|------|-------|-------|--------|
| Dominant (60%) | `#0a0a0f` | Full-page background, grid background layer, card backgrounds | CONTEXT.md code_context — App.tsx already applies this |
| Secondary (30%) | `#0d1117` | Card surface (1 shade lighter than dominant), header bar background, Now Playing banner background | Default — matches near-black dark panel pattern |
| Accent — Healthy (10%) | `#00c8ff` (Tron Blue) | Online status dot, healthy card border trace, healthy card glow, section label text, primary UI text, app title | REQUIREMENTS.md DASH-05 |
| Accent — Warning | `#ffaa00` (Amber) | Warning status dot, warning card border trace, warning card pulse glow, stale-data indicator badge | REQUIREMENTS.md DASH-05 |
| Accent — Critical | `#ff4444` (Red) | Offline/critical status dot, critical card border trace, critical card glow flash | REQUIREMENTS.md DASH-05 |
| Destructive | `#ff4444` | Same as Critical — no destructive actions exist in Phase 2 (no delete/reset); reserved for Phase 3+ | Default |
| Muted text | `#4a5568` | Placeholder metric slots ("—"), stub page body text, disabled states | Default — 60% lightness-reduced version of dominant |
| Grid line color | `rgba(0, 200, 255, 0.06)` | Animated background grid lines at rest | Claude's Discretion — subtle, not distracting |
| Grid pulse color | `rgba(0, 200, 255, 0.18)` | Traveling pulse highlight as it sweeps along grid lines | Claude's Discretion |
| Section label color | `rgba(0, 200, 255, 0.45)` | Section tier labels (STATUS, ACTIVITY, etc.) — dimmed Tron Blue, not full brightness | Claude's Discretion |

**Accent reserved for:** online status dots, card border traces, card glow effects, section labels, the app title "CORUSCANT", and the Now Playing banner expand indicator. Never use Tron Blue on generic button backgrounds or as a fill color for cards.

**Card surface opacity:** Cards use `background: rgba(13, 17, 23, 0.85)` with `backdrop-filter: blur(4px)` so the animated grid shows through subtly without overwhelming card content.

**Color for mock data:** The NAS stats strip (CPU, RAM, disk) uses Tron Blue at 100% opacity for values, muted text for labels.

## Animation Contract

This section is mandatory for this phase due to REQUIREMENTS.md DASH-02, DASH-03, DASH-04, DASH-08.

**Rule: All animations must use `transform` and `opacity` only. No `height`, `width`, `margin`, `top`, `left`, `color`, or `background-color` in `@keyframes` bodies. (DASH-08)**

### Grid Background (CSS-only, D-01)

| Property | Value | Notes |
|----------|-------|-------|
| Grid line width | 1px | |
| Grid cell size | 48px | Feels like Tron grid at mobile viewport widths |
| Grid color | `rgba(0, 200, 255, 0.06)` | See color table above |
| Pulse sweep direction | horizontal (left to right) + vertical (top to bottom) on alternating columns/rows | Use two separate pseudo-elements |
| Pulse duration | 8s per sweep | Slow enough to feel ambient |
| Pulse easing | `ease-in-out` | |
| Pulse opacity range | 0 → 0.18 → 0 | See color table |
| `position` | `fixed`, `inset: 0`, `z-index: 0` | Cards scroll over it (D-02) |
| `pointer-events` | `none` | Grid must never intercept clicks |
| Animation intensity CSS var | `--grid-pulse-opacity: 0.18` | Phase 2 Settings stub reads/writes this var; default 0.18 |
| `will-change` | `opacity` | On the pseudo-element pulse only |

### Card Border Traces (CSS-only or Framer Motion, D-11)

| Property | Value | Notes |
|----------|-------|-------|
| Implementation | CSS `@keyframes` on `::before` pseudo-element | Prefer CSS for performance on ARM; Framer Motion for health-state variant switching if CSS fallback proves complex |
| Trace color | Matches card health state (Tron Blue / Amber / Red) | |
| Trace opacity at rest | 0.5 | Not full brightness |
| Duration | 3s per full loop | |
| Stagger per card | 0.3s phase offset per card index | Apply via `animation-delay: calc(var(--card-index) * 0.3s)` and `--card-index` CSS custom property set inline |
| Loop | `infinite` | Continuous, no event trigger |
| `will-change` | `opacity, transform` | |

### Card Health-State Glow (CSS-only, D-04)

| Health State | Animation | Duration | `box-shadow` |
|-------------|-----------|----------|--------------|
| `online` (healthy) | `@keyframes breathe` — opacity 0.3 → 0.7 → 0.3 | 3s ease-in-out infinite | `0 0 12px 2px rgba(0, 200, 255, var(--glow-opacity))` |
| `warning` | `@keyframes pulse-amber` — opacity 0.4 → 0.85 → 0.4 | 1.2s ease-in-out infinite | `0 0 16px 3px rgba(255, 170, 0, var(--glow-opacity))` |
| `offline` / `critical` | `@keyframes flash-red` — opacity 0.2 → 1.0 → 0.2 | 0.7s ease-in-out infinite | `0 0 20px 4px rgba(255, 68, 68, var(--glow-opacity))` |
| `stale` | `@keyframes breathe` at reduced opacity | 4s ease-in-out infinite | Same as healthy but dimmed — `var(--glow-opacity)` = 0.2 |

**Note:** `box-shadow` changes between states are not animated (that would cause layout recalculation). Instead, all four `box-shadow` declarations are defined as separate CSS classes and the health-state class is toggled. The `@keyframes` only animate `opacity` of the element.

### Card Entrance Animation (Framer Motion, recommended)

Use Framer Motion `motion.div` with `initial={{ opacity: 0, y: 12 }}` → `animate={{ opacity: 1, y: 0 }}` on initial mount. Stagger via `delay: index * 0.05`. Duration: 0.3s. Easing: `easeOut`. This only fires once on page load — it does not re-trigger on SSE updates.

### Now Playing Banner Slide (Framer Motion, D-16, D-17)

| Transition | Implementation | Values |
|------------|---------------|--------|
| Hidden → collapsed strip | `AnimatePresence` with `motion.div`; `initial={{ y: 48, opacity: 0 }}` → `animate={{ y: 0, opacity: 1 }}` | Duration: 0.25s, ease: easeOut |
| Collapsed → expanded drawer | `motion.div` height: fixed 48px → expanded height via `layout` prop | Duration: 0.3s, ease: easeInOut |
| Expanded → collapsed | Reverse of above | Duration: 0.25s, ease: easeIn |
| Unmount (no streams) | `exit={{ y: 48, opacity: 0 }}` | Duration: 0.2s |

**Ticker scroll (D-15):** Implement as CSS `@keyframes marquee` on an inner `<span>` — `transform: translateX(0)` → `transform: translateX(-50%)`. Pause animation when banner is expanded. Only animate when text overflows container width (check via `scrollWidth > clientWidth` in a `useEffect`).

### AppHeader dimensions

- Total height: 88px (two rows of 44px each)
- Title row: 44px height, horizontal padding 16px
- NAS stats strip: 44px height, horizontal padding 16px
- Font: Body (14px) for stat values, Label (12px) for stat keys

### Service Card dimensions

- Min width: 160px (grid `minmax(160px, 1fr)`)
- Height: 88px (matches header height — feels intentional)
- Internal padding: 12px (`card-inner` token — see Spacing Scale)
- Card border radius: 6px
- Border: 1px solid `rgba(0, 200, 255, 0.15)` at rest; trace animation brightens this via `::before` overlay

### Scroll behavior

- `main` content area: `padding-top: 88px` (clears fixed header), `padding-bottom: 64px` (clears fixed banner + safe area)
- Scroll position save on card tap: `sessionStorage.setItem('dashboardScrollY', window.scrollY.toString())` before navigation; restore on back via `window.scrollTo(0, savedY)` in `useEffect` on mount.

## Interaction Contract

| Interaction | Trigger | Response | Animation |
|-------------|---------|----------|-----------|
| Tap service card | `onClick` / touch | Navigate to `/services/:serviceId` via React Router `useNavigate` | Card scales to `0.97` on press (`transform: scale(0.97)`, duration 0.1s) then releases; Framer Motion `whileTap` |
| Browser back from detail | Back gesture / button | Return to `/`; restore scroll position from `sessionStorage` | No custom animation — browser handles transition |
| Tap Now Playing banner (collapsed) | `onClick` on banner strip | Expand drawer | Framer Motion slide-up (see Animation Contract) |
| Tap outside expanded drawer | `onClick` on overlay backdrop | Collapse drawer | Framer Motion slide-down |
| Tap expanded drawer again | `onClick` on banner strip | Collapse drawer | Framer Motion slide-down |
| SSE `dashboard-update` event | EventSource message | Replace full state via `setSnapshot(newSnapshot)` | No transition on data update — values update in place; status dot color change is instant |
| SSE connection lost | `EventSource.onerror` | Set `connected: false`; show stale amber dot in header; auto-reconnect via `new EventSource('/api/sse')` after 5s | No full-page state change |
| Header icon button tap (⚙) | `onClick` | Navigate to `/settings` | No animation |
| Header icon button tap (≡) | `onClick` | Navigate to `/logs` | No animation |

## Accessibility Contract

| Concern | Requirement |
|---------|-------------|
| Color contrast | Tron Blue `#00c8ff` on `#0a0a0f` background: contrast ratio ≈ 8.4:1 — passes WCAG AA (4.5:1) and AAA (7:1). Amber `#ffaa00` on `#0a0a0f`: ≈ 6.7:1 — passes AA. Red `#ff4444` on `#0a0a0f`: ≈ 4.6:1 — passes AA. |
| Touch targets | All interactive elements ≥ 44×44px (iOS HIG). Header icon buttons use `width: 44px; height: 44px; display: flex; align-items: center; justify-content: center` to achieve the minimum without non-multiple-of-4 padding. Service cards at 160px min-width × 88px height exceed this. |
| Animation preference | Wrap all `@keyframes` and Framer Motion animations in `@media (prefers-reduced-motion: reduce)` fallback: set `animation-duration: 0.001ms !important` for CSS; pass `transition={{ duration: prefers ? 0 : 0.3 }}` for Framer Motion via a hook. |
| Screen reader | Service cards include `aria-label="{name}, status: {status}"`. Status dot is `aria-hidden="true"` (color-only indicator). Now Playing banner has `aria-live="polite"` for stream count updates. Header Settings icon button has `aria-label="Open Settings"`. Header Logs icon button has `aria-label="Open Logs"`. |
| Focus management | After navigating from card to detail, focus moves to the `<h1>` of the detail page. On back, focus returns to the card that was tapped (store card ID in `sessionStorage` alongside scroll position). |
