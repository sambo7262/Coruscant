  <action>
Add a barely-perceptible slow drift to the background to prevent burn-in on the always-on kiosk display.

**Approach — CSS transform drift (GPU-composited, zero layout/paint cost):**

Add a `@keyframes spaceFloat` animation to the GridBackground wrapper element:
```css
@keyframes spaceFloat {
  0%   { transform: translate(0px, 0px); }
  25%  { transform: translate(2px, -1px); }
  50%  { transform: translate(1px, 2px); }
  75%  { transform: translate(-2px, 1px); }
  100% { transform: translate(0px, 0px); }
}
```

Apply to the GridBackground root element:
```css
animation: spaceFloat 90s ease-in-out infinite;
will-change: transform;
```

**Key constraints:**
- Total movement: max ±3px in any direction — imperceptible at kiosk distance but enough to shift pixels
- Duration: 90 seconds — very slow cycle that users won't notice consciously
- `ease-in-out` on each keyframe segment — no snapping or jarring transitions
- `will-change: transform` — ensures GPU compositing, no CPU involvement
- The GridBackground is `position: fixed; inset: 0` and sits behind everything — shifting it 3px is invisible to users but exercises all background pixels

**Additionally:** Add a slow opacity breathe to the nebula `::before` pseudo:
```css
@keyframes nebulaBreath {
  0%, 100% { opacity: 0.6; }
  50% { opacity: 1.0; }
}
/* Apply to ::before: animation: nebulaBreath 120s ease-in-out infinite; */
```
This makes the faint blue glow very slowly shift in intensity — like the cockpit floating through a nebula.

**Do NOT animate:** card elements, text, LEDs, or anything interactive. Only the fixed background layer.
  </action>
