**Step 2 — Create packages/backend/src/adapters/unifi.ts:**

Follow the pihole.ts/nas.ts adapter pattern. Key implementation details:

- `const TIMEOUT_MS = 10_000`
- Module-level variables: `let cachedSiteId: string | null = null`, `let peakTxMbps = 0`, `let peakRxMbps = 0`
- `schedulePeakReset()` using `setTimeout` with `PEAK_WINDOW_MS = 6 * 60 * 60 * 1000` (6 hours). Call once at module init. Reset nulls peaks and restarts timer. Also export a `_peakResetTimerRef` or use `let peakResetTimer: ReturnType<typeof setTimeout> | null` so tests can clear it.
- Export `function resetUnifiCache(): void` that sets `cachedSiteId = null`, `peakTxMbps = 0`, `peakRxMbps = 0`
- Export helper functions for testability: `classifyModel(model: string): 'gateway' | 'switch' | 'ap' | 'unknown'`, `computeHealthStatus(devices: UnifiDevice[]): 'online' | 'warning' | 'offline'`, `formatUptime(seconds: number): string`

`classifyModel` logic (per D-06):
```typescript
const m = model.toUpperCase()
if (m.startsWith('UDM') || m.startsWith('UDMP') || m.startsWith('UDR')) return 'gateway'
if (m.startsWith('USW')) return 'switch'
if (m.startsWith('U6') || m.startsWith('UAP') || m.startsWith('UAL') || m.startsWith('UAE')) return 'ap'
return 'unknown'
```

`computeHealthStatus` logic (per D-05 — gateway-first):
```typescript
const gateways = devices.filter(d => classifyModel(d.model) === 'gateway')
const others = devices.filter(d => classifyModel(d.model) !== 'gateway')
if (gateways.length === 0 || gateways.some(d => d.state !== 'online')) return 'offline'
if (others.some(d => d.state !== 'online')) return 'warning'
return 'online'
```

`formatUptime` logic:
```typescript
const days = Math.floor(seconds / 86_400)
const hours = Math.floor((seconds % 86_400) / 3_600)
if (days > 0) return `${days}d ${hours}h`
return `${hours}h`
```

`resolveSiteId(baseUrl, apiKey)`:
- If `cachedSiteId` is not null, return it immediately
- `GET ${baseUrl}/proxy/network/integration/v1/sites` with headers `{ 'X-API-KEY': apiKey }` and `timeout: TIMEOUT_MS`
- Response shape: `{ data: [{ siteId: string, internalId: string, name: string }] }`
- Match on `internalId === 'default'` first, fall back to `name.toLowerCase() === 'default'`, fall back to `sites[0]`
- Cache in `cachedSiteId` and return

`pollUnifi(baseUrl: string, apiKey: string): Promise<ServiceStatus>`:
- Wrap entire body in try/catch. On any error, return `{ id: 'unifi', name: 'UniFi', tier: 'rich', status: 'offline', configured: true, lastPollAt: new Date().toISOString() }`
- Call `resolveSiteId(baseUrl, apiKey)` to get siteId
- Make three parallel requests with `Promise.all`:
  1. `GET ${baseUrl}/proxy/network/integration/v1/sites/${siteId}/devices` with `X-API-KEY` header
  2. `GET ${baseUrl}/proxy/network/integration/v1/sites/${siteId}/clients` with `X-API-KEY` header
  3. `GET ${baseUrl}/proxy/network/api/s/default/stat/health` with `X-API-KEY` header — wrap in its own try/catch so 401 returns null instead of failing the whole poll
- Parse devices: map each device from API response to `UnifiDevice` shape: `{ macAddress: d.macAddress, model: d.model, name: d.name, state: d.state, uptime: d.uptime ?? 0, clientCount: d.features?.access_point?.num_sta ?? 0 }`
- Parse clients: `totalCount` from response for aggregate client count
- Parse WAN: find subsystem `'wan'` in stat/health data array. Access `wan['tx_bytes-r']` and `wan['rx_bytes-r']` using bracket notation (field name has literal hyphen). Convert bytes/s to Mbps: `value / 125_000`. If stat/health call failed (returned null), set `wanTxMbps: null` and `wanRxMbps: null`
- Update peaks: `if (txMbps !== null && txMbps > peakTxMbps) peakTxMbps = txMbps` (same for rx)
- Compute healthStatus via `computeHealthStatus(devices)`
- Return `ServiceStatus` with `id: 'unifi'`, `name: 'UniFi'`, `tier: 'rich'`, `status` mapped from healthStatus ('online' stays 'online', 'warning' stays 'warning', 'offline' stays 'offline'), `configured: true`, `metrics` cast as `UnifiMetrics as unknown as Record<string, unknown>`
